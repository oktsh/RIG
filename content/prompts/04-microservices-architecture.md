Архитектурные паттерны для масштабируемых распределённых систем.

## Промпт

Спроектируйте архитектуру микросервисов с учетом масштабируемости, отказоустойчивости и независимого развертывания. Определите границы сервисов и коммуникационные паттерны.

## Когда использовать

- При переходе от монолита к микросервисам
- При проектировании новой распределённой системы
- При масштабировании существующей архитектуры
- При планировании инфраструктуры для высоконагруженных сервисов

## Рекомендуемые технологии

- Kubernetes -- для оркестрации контейнеров
- Docker -- для контейнеризации сервисов

## Пример использования

```
Спроектируйте архитектуру микросервисов для [описание системы].

Определите:
1. Границы сервисов (bounded contexts)
2. Коммуникационные паттерны:
   - Синхронные (REST / gRPC)
   - Асинхронные (события, очереди)
3. Стратегию данных:
   - Database per service
   - Saga паттерн для распределённых транзакций
4. Инфраструктуру:
   - Service mesh
   - API Gateway
   - Circuit breaker
5. Стратегию развёртывания:
   - CI/CD пайплайн
   - Blue-green / canary деплой
   - Мониторинг и алертинг
```
