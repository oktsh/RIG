Автоматическое ревью с фокусом на безопасность, производительность и стандарты команды.

## Промпт

Проанализируйте код на предмет безопасности, производительности, соответствия стандартам команды и лучшим практикам. Предложите конкретные улучшения с примерами.

## Когда использовать

- При ревью pull request перед мержем
- При аудите безопасности существующего кода
- При оптимизации производительности критичных участков
- При онбординге новых разработчиков для проверки кода

## Рекомендуемые инструменты

- Cursor -- для встроенного ревью прямо в IDE
- Copilot -- для автоматических предложений по улучшению

## Пример использования

```
Проведите код-ревью следующего файла. Проверьте:

1. Безопасность: SQL-инъекции, XSS, утечки данных
2. Производительность: N+1 запросы, лишние ререндеры, утечки памяти
3. Стандарты: именование, структура, типизация
4. Лучшие практики: SOLID, DRY, обработка ошибок

Для каждой проблемы укажите:
- Серьёзность (критичная / важная / рекомендация)
- Строку кода
- Описание проблемы
- Пример исправления
```
